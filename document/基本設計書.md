# 基本設計書（講師提供）

> フェーズB の詳細設計書作成・実装のベースとなる設計書です。
> この設計書の内容を読み解き、詳細設計書（workフォルダ）に落とし込んでください。

---

## 1. テーブル定義

### customers（顧客マスタ）

| カラム名 | 型 | 制約 | 備考 |
|---------|---|------|-----|
| id | BIGINT | PK, AUTO INCREMENT | 自動採番されるため、登録時に指定不要 |
| customer_name | VARCHAR(100) | NOT NULL | 顧客名 |
| phone | VARCHAR(20) | NULL | 電話番号 |
| postal_code | VARCHAR(8) | NULL | 郵便番号（例: 100-0001）|
| address | VARCHAR(200) | NULL | 住所 |
| email | VARCHAR(255) | NULL | メールアドレス |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 登録日時（@PrePersist で自動設定） |

### products（商品マスタ）

> `unit_price`（単価）は受注時に order_details テーブルにコピーされます（フェーズC で使用）。

| カラム名 | 型 | 制約 | 備考 |
|---------|---|------|-----|
| id | BIGINT | PK, AUTO INCREMENT | 自動採番されるため、登録時に指定不要 |
| product_name | VARCHAR(100) | NOT NULL | 商品名 |
| category | VARCHAR(50) | NULL | 商品分類（選択肢は4章を参照） |
| unit_price | INTEGER | NOT NULL, CHECK >= 1 | 単価（税抜き）|
| stock_quantity | INTEGER | NOT NULL, DEFAULT 0, CHECK >= 0 | 在庫数 |

## 2. 画面一覧

### 顧客マスタ

| 画面名 | URL | HTTP メソッド | 説明 |
|-------|-----|-------------|-----|
| 顧客一覧 | /customers | GET | 顧客の一覧表示・名前で部分一致検索（`?name=xxx`） |

| 顧客編集フォーム | /customers/{id}/edit | GET | 顧客編集フォーム表示 |
| 顧客更新処理 | /customers/{id} | POST | 顧客更新実行 |
| 顧客削除処理 | /customers/{id}/delete | POST | 顧客削除実行 |

### 商品マスタ

| 画面名 | URL | HTTP メソッド | 説明 |
|-------|-----|-------------|-----|

| 商品登録フォーム | /products/new | GET | 商品登録フォーム表示 |
| 商品登録処理 | /products | POST | 商品登録実行 |

## 3. クラス構成（顧客マスタ）

> 各クラスの役割を把握してから実装を始めましょう。
> 商品マスタも同じ構成で作成します（`Customer` → `Product` に読み替える）。

```
controller/
  CustomerController.java   … 画面遷移・フォーム受取（URL のルーティングを担当）

service/
  CustomerService.java      … ビジネスロジック・バリデーション後処理（業務ルールを担当）

repository/
  CustomerRepository.java   … JpaRepository を継承・検索クエリ定義（DB 操作を担当）

entity/
  Customer.java             … customers テーブルに対応する Entity（DB の1行 = 1オブジェクト）

form/
  CustomerForm.java         … 登録・編集フォームの入力値を保持するクラス（画面↔Controller 間のデータ受け渡し）
```

## 5. バリデーション仕様

### 顧客マスタ

| フィールド | バリデーション |
|----------|------------|
| customer_name | 必須、100文字以内 |
| phone | 任意、10〜13桁の数字とハイフン |
| postal_code | 任意、`^\d{3}-\d{4}$` 形式 |
| address | 任意、200文字以内 |
| email | 任意、メール形式（`@Email`） |

### 商品マスタ

| フィールド | バリデーション |
|----------|------------|
| product_name | 必須、100文字以内 |
| unit_price | 必須、1以上 9,999,999 以下 |
| stock_quantity | 必須、0以上 |

---

## 6. エラー処理方針

| 状況 | 対応 |
|-----|-----|
| バリデーションエラー | 同画面に戻りエラーメッセージを表示（`BindingResult` のエラーを Thymeleaf で表示） |
| 存在しない ID へのアクセス | `ResourceNotFoundException` をスローし `/error` へ遷移（例：`/customers/99999/edit`） |
| その他サーバーエラー | `/error` へ遷移しエラーメッセージを表示 |
